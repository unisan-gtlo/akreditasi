1. Kelola Sesi Akreditasi

Sesi adalah "proyek akreditasi" - misal "Re-akreditasi Manajemen LAMEMBA 2025" atau "Akreditasi Baru Teknik Elektro BAN-PT 2026".

1Menu sidebar Sesi Akreditasi - list semua sesi yang pernah dibuat
2Klik + Buat Sesi Baru, isi form:
  • Judul - deskriptif, misal "Re-akreditasi Manajemen LAMEMBA 2026"
  • Instrumen - pilih LAMEMBA/BAN-PT/dll
  • Kode Prodi - misal E21 (opsional, untuk akreditasi prodi)
  • Tahun TS - tahun akademik aktif sesi
  • Tanggal Target - deadline submission
3Simpan - sesi terbuat dengan status Persiapan

2. Timeline & Milestone

Setiap sesi punya timeline untuk tracking progres.

1Buka detail sesi, klik tombol Timeline (biru)
2Lihat milestone default yang ter-generate (Persiapan, Upload, Review LP3M, Submission, Visitasi, Sertifikat)
3Klik Edit Milestone untuk tanggal target & catatan

3. Bundle Export

Bundle Export menggabungkan semua dokumen sesi dalam 1 tampilan terstruktur (Standar → Sub-Standar → Butir → Dokumen).

1Buka detail sesi, klik tombol Bundle (ungu)
2Halaman bundle tampilkan:
  • Stats: total butir, terisi, kosong, dokumen, progress %
  • Tree hierarkis Standar → SubStandar → Butir
  • Status verifikasi per dokumen (badge warna-warni)
3Pakai search bar untuk filter butir tertentu, atau collapse/expand section
4Klik Print untuk print-friendly view (otomatis expand semua, hide toolbar)

4. Token Share untuk Asesor

Share bundle ke asesor LAM/BAN-PT tanpa perlu mereka login.

1Di halaman bundle, scroll ke section Kelola Token Share (kuning)
2Isi form:
  • Label - keperluan token, misal "Asesor Dr. Budi LAMEMBA"
  • Expired (opsional) - tanggal kadaluarsa link
3Klik + Buat Token
4Token link muncul di daftar. Klik Copy - paste ke email/WA asesor
5Asesor buka link di browser - langsung lihat bundle TANPA login (dengan watermark "BUNDLE ASESOR")
Penting: Bundle Public hanya tampilkan dokumen berstatus Approved. Pastikan semua dokumen sudah di-verifikasi LP3M sebelum share token.

Revoke Token

Kalau asesor sudah selesai atau link bocor, klik tombol Revoke pada token tersebut. Link langsung tidak bisa diakses.

5. Export ZIP

Download semua dokumen bundle dalam 1 file ZIP untuk submission offline atau arsip.

1Di halaman bundle, klik tombol Export ZIP (ungu)
2Browser download file bundle_[INSTRUMEN]_[PRODI]_[TAHUN]_[TANGGAL].zip
3Isi ZIP:
  • README.txt - info sesi + stats
  • manifest.json - metadata semua dokumen
  • Folder hierarkis Standar/SubStandar/Butir/ berisi file dokumen
Catatan: File yang disimpan di Google Drive tidak disertakan langsung di ZIP (biar tidak besar), tapi link aksesnya tercatat di manifest.json.

6. Monitoring & Laporan

  • Dashboard utama - summary semua sesi aktif dengan progress
  • Django Admin - akses raw database untuk audit (hanya superuser)
  • Log Verifikasi - history lengkap siapa approve/reject apa kapan (Django Admin → Log Verifikasi)
  • Log Akses Dokumen - track siapa yang pernah buka dokumen sensitif
